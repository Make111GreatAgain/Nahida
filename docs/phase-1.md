# Nahida Phase 1

Phase 1 is a no-code setup:

1. Use `vault/` as the starting Obsidian vault.
2. Read `vault/00_System/NAHIDA_SPEC.md`.
3. Use `skills/nahida-intake` for one web source.
4. Use `skills/nahida-distill` to create one claim.
5. Use `skills/nahida-map-update` to link the claim from a map.
6. Use `skills/nahida-audit` to produce an audit report.

Code is intentionally absent from Phase 1. Automation scripts can be added later as bundled skill resources only when a repeated operation needs deterministic execution.
