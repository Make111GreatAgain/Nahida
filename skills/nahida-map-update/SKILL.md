# Nahida Map Update

Read `vault/00_System/NAHIDA_SPEC.md` before writing.

## Workflow

1. Identify target map and allowed section.
2. Read relevant sources, claims, concepts, bridges, and direction candidates.
3. Update machine-owned sections such as `Recent Updates`, `Watchlist`, or `Emerging Directions`.
4. Keep entries linked to source or claim IDs.
5. Send structural taxonomy changes to review if confidence is low.

## Boundaries

- May write managed `07_Maps/` sections.
- Must not create source, claim, or bridge notes.
