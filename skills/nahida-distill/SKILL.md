# Nahida Distill

Read `vault/00_System/NAHIDA_SPEC.md` before writing.

## Workflow

1. Read the source note and its provenance.
2. Extract atomic claims with clear evidence.
3. Mark claims as `tentative`, `supported`, `contested`, or `retired`.
4. Link every claim to source IDs.
5. Update draft concept sections only in managed notes.
6. Send uncertain synthesis to review queues.

## Boundaries

- May write `04_Claims/`, draft sections in managed `05_Concepts/`, and review queues.
- Must not edit raw artifacts or source identity fields.
