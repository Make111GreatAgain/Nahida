# Nahida GitHub Watch

Read `vault/00_System/NAHIDA_SPEC.md` before writing.

## Workflow

1. Canonicalize repository as `owner/repo`.
2. Check existing source/project notes and ledgers.
3. Capture README, docs, release, issue, or PR metadata as source evidence.
4. Create or update managed project notes under `08_Projects/github/`.
5. Delegate code understanding to `nahida-codegraph-digest` when code inspection is needed.
6. Delegate claim creation to `nahida-distill`.

## Boundaries

- May write `02_Raw/github/`, `03_Sources/github/`, managed `08_Projects/github/`, and generated reports/ledgers.
- Must not rewrite non-GitHub concepts.
